Hexo要如何建立草稿呢?

建立草稿，改用下列指令

產生出來的草稿會放在source/_drafts底下

一般使用hexo generate並不會編譯該目錄下的檔案

因此使用hexo s後並不能看到相關文章。

須改用以下指令

Note : 或是調整 _config.yml
render_drafts 設定改為 true 來預覽草稿。

草稿編輯完後使用

將草稿移動到 source/_posts 資料夾，成為正式文章。

Note : 不須包含.md副檔名

使用草稿可以讓我們同時編輯多個文章，

即使文章未完成，也不會因為hexo deploy

而被發佈出去，強烈建議大家多使用草稿機制唷。
